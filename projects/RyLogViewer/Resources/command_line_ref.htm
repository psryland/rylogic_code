Command Line Syntax

Syntax:rylogviewer [filepath] [options]

Following are the valid command line options for RylogViewer. These options may be given in any order.

OptionMeaning
"file path" The path to a file to load on startup.

Example:
rylogviewer "mylogfile.log"
-p Portable Mode.
Portable mode allows the application to run from any location that allows file write access, e.g. a USB drive. In portable mode, settings are written to the same directory that the application is running in. If a file with the name 'portable' is found in the same directory as the executable then portable mode will be enabled by default.
Note: that portable mode is equivalent to: rylogviewer -s ".\settings.xml"

Example:
rylogviewer -p
-s"settings file path" Loads the log viewer using the specified settings file path.
This option is useful for running multiple instances of the log viewer without overwriting the settings of other instances.

Example:
rylogviewer -s"C:\temp\mysettings.xml"
-e"export file path" This option instructs the log viewer to export a file to "export file path" using the current filter and transform settings. "file path" must also be given on the command line. It is often useful to use the '-fx' and '-tx' options in addition to this option. The application main window is not displayed during the export and, on completion, the returned exit code is '0' for success, '1' for failure.

Examples:
rylogviewer ALogFile.log -e"C:\exported logs\ALogFile.Exported.txt"
rylogviewer ALogFile.log -ft"filters.rylog_filters" -tx"transforms.rylog_transforms" -e"C:\exported logs\ALogFile.Exported.txt"
-rdelim"row delimiter" This option specifies the row delimiter to use when exporting a file using the '-e' option. If not given, the row delimiters from the input file are used. If the -e option is not used, this option is ignored. Use the following special strings to represent carrage return, line feed, and tab:
  • <CR> - carrage return
  • <LF> - line feed
  • <TAB> - tab
Examples:
rylogviewer ALogFile.log -e"C:\exported logs\ALogFile.Exported.txt" -rdelim"<CR><LF>"
-cdelim"column delimiter" This option specifies the column delimiter to use when exporting a file using the '-e' option. If not given, the column delimiters from the input file are used. If the -e option is not used, this option is ignored. Use the following special strings to represent carrage return, line feed, and tab:
  • <CR> - carrage return
  • <LF> - line feed
  • <TAB> - tab
Examples:
rylogviewer ALogFile.log -e"C:\exported logs\ALogFile.Exported.txt" -cdelim"<TAB>"
-hl"highlight set file path" Loads the specified highlight set, replacing any currently configured highlights

Example:
rylogviewer -hl"myhighlights.rylog_highlights"
-ft"filter set file path" Loads the specified filter set, replacing any currently configured filters

Example:
rylogviewer -ft"myfilters.rylog_filters"
-tx"transform set file path" Loads the specified transform set, replacing any currently configured transforms

Example:
rylogviewer -tx"mytransforms.rylog_transforms"
-nogui Suppresses the displaying of any gui (e.g. progress dialogs) while running from the command line.
-h Displays this help message.
