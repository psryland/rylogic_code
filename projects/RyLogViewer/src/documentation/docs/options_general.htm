General Tab
Under the general tab you will find controls for the startup, file loading, and file parsing behaviour of RyLogViewer. Click on the yellow boxes in the image below to jump to a description of each control.
  1. Settings File Location
    This is the full file path for the settings file that RyLogViewer is currently using. This field is informational only. To change the settings file path, use the load and save as settings buttons.
  2. Reset Settings
    Use this button to reset data in the settings file to the default values.
  3. Load Settings
    Use the load button to load an alternate settings file. Once loaded, that file is then used for subsequent settings changes.
  4. Save Settings
    Use the save button to save a copy of the current settings to a new location. Once saved, the new file location is used for subsequent settings changes.
  5. Load Last File
    Check this option to have RyLogViewer load the last opened log file on startup. Useful if you use RyLogViewer to view the same log file regularly.
  6. Restore Screen Position
    Checking this option will cause RyLogViewer to open at the screen location that it was last in when closed.
  7. Show Tip of the Day
    Check this option to have the 'Tip of the Day' dialog windows show when RyLogViewer opens.
  8. Check for Updates
    Check this option to have RyLogViewer contact the Rylogic Limited server and check for the release of a newer version.
  9. Use Web Proxy
    If you are using RyLogViewer on a domain that uses a proxy server, select this option to specify the proxy server for web connections. (Used mainly for the check for updates feature).
  10. Proxy Host
    The IP or domain name of the proxy server
  11. Proxy Port
    The port to use when communicating with the proxy server
  12. Cached Lines Count
    A key feature of RyLogViewer is it's ability to load files of any size, limited only by the operating system. This is achieved by only ever loading part of the log file into memory at any one time. The value chosen here for the number of cached log file lines affects when RyLogViewer needs to load data from the log file. Specifying a larger number for this option reduces how often RyLogViewer needs to load data, but increases the loading time for reload operations (caused by jumping around in the file), and the amount of memory used.

    When choosing a value for this setting it is best to experiment with a range of values. If loading a log file that is located on a network shared drive, often a smaller value improves loading performance.
  13. Maximum Data Loaded
    This setting works in conjunction with the Cached Lines Count setting. Since the number of lines in a log file is not directly related to the actual size of the data loaded, it is necessary to set an upper limit on the amount of data loaded. (This limit also prevents "run-away" file loading if the Line Endings settings is incorrect for the file being loaded).

    RyLogViewer will load the number of lines given by the Cached Lines Count setting or the number of bytes specified in the Maximum Data Loaded setting, whichever limit occurs first.
  14. Maximum Line Length
    This settings defines the upper limit on how long any one single line can be in the log file. RyLogViewer does not display truncate lines. If this limit is ever reached an error message will be displayed instructing the user to increase the limit or, as is more often the case, check the Line Endings settings. An incorrect line ending setting can cause RyLogViewer to miss the line endings and assume that the entire log file is on a single line!
  15. Open At End
    Check this option to have RyLogViewer open and display the last line in the log file. Typically used for log files that are changing while RyLogViewer is displaying the file.
  16. Assume Additive File Changes
    This setting affects the file loading performance of RyLogViewer. When checked, RyLogViewer will assume that any changes to the log file only include data being added to the file. Any data that RyLogViewer already has cached is assumed to still be valid. If the setting is unchecked, this assumption is not made, and file changes will result in all cached data being reloaded. This setting can have a significant effect on loading performance and the Cached Lines Count setting may need reducing if this option is unchecked.
  17. Ignore Blank Lines
    If checked, RyLogViewer will not display blank lines. The same effect can be achieved with a filter, however selecting this option is often quicker and simpler.
  18. Custom Line Ending
    The options available in the Line Endings menu include the option for a custom line ending. This field is where the custom line ending is defined. Any text can be used in this field. So, for example, a comma separated list could be interpreted as rows by using ',' as the custom line ending. The special character sequences; <CR>, <LF>, and <TAB> are interpreted as carriage return, line feed, and the tab character respectively (\u000D, \u000A, and \u0009).
  19. Column Delimiters
    If specified, RyLogViewer will divide lines in the log file into columns, delimited by the characters given in this setting. If left blank, RyLogViewer will use a single column that contains the entire line.
  20. Column Count
    This setting is used to limit the number of columns displayed. Only used when the Column Delimiters field is not empty.
