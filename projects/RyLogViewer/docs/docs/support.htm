F.A.Q.
RyLogViewer does not display modified files correctly
Check the state of Additive File Changes mode. (The green 'plus' button on the tool bar).
When enabled, RyLogViewer assumes that data will only ever be added to the viewed file and so any cached data it has will remain valid. This greatly improves performance for files or streamed data that are appended to only.

However, if data in the file can be changed anywhere within the file, then Additive File Changes mode must be disabled. This will cause RyLogViewer to dump any cached data, ensuring correct file data is always displayed.

A frequent cause of corrupted file data being displayed occurs when Additive File Changes mode is enabled, but the file has been changed in a way other than by appending data.
Other programs that use adb.exe disconnect when RyLogViewer starts
Check whether you have more than one instance of adb.exe installed on your system and verify that RyLogViewer is running the same version of adb.exe as the other programs. Running different versions of adb.exe can cause the ADB service to reset which is usually the cause of existing adb.exe instances disconnecting.

To find out which version of adb.exe RyLogViewer is running, open the Android Logcat wizard.

RyLogViewer's use of adb.exe is no different to running adb.exe from a command prompt, so to help diagnose issues like this, try running adb.exe from a command prompt window.
