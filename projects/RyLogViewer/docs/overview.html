Overview and Features
Many computer systems output log data in various forms. From servers logging connection events, to embedded systems streaming real-time state information, log output often contains vital diagnostic information. Frequently this vital information is hidden within large volumes of general output making finding, analysing, and diagnosing issues in the computer system a difficult and time consuming task. RyLogViewer was written to make working with high-volume log data easier.

The simplest but probably most useful feature of RyLogViewer, is the ability to highlight lines in the log data based on some pattern, the canonical example being to have lines containing the word error appear with a red background.

Patterns can be as simple as "any line that contains the word 'error' " to complex regular expressions. Once familiar with the pattern system, the same knowledge can be applied to search for specific text in the log data, create filters to strip out unwanted noise, transform and rearrange text into more meaningful/human- readable forms, and even setup actions to perform when double clicking on lines in the log data. (Check out the ideas section for inspiration on ways in which RyLogViewer can be used).

RylogViewer is a shareware application, meaning there are limits applied to the free version of the application that are removed in the licensed version. However, these limits do not restrict or disable any of the functionally of the application. The free version merely prompts the user when a limit is exceeded giving the user the option to enable the licensed feature. After minutes, the prompt will reappear. There is no time restriction or limit on the number of times licensed features can be enabled.

The bullet points below give a summary of the main features and functionality of RyLogViewer, including links to further information in the remainder of the documentation.
  • Unlimited File Sizes
    There is no upper limit on the size of displayed log files. RyLogViewer has been designed to intelligently spool and cache data dynamically, meaning that it runs as smoothly on a 200Gb file as it does on a 2Kb file.
  • Highlighting
    Limited in Free Edition
    Important data in a log file can be made to stand out by creating a pattern in the highlighting options window. Highlighting can be quickly enabled or disabled via the tool bar. In the free version, highlighting is limited to patterns, however more patterns can be used by selecting enable in the Free Edition Limited Feature dialog.
  • Filtering
    Limited in Free Edition
    Patterns can be created in the filtering options window to strip unwanted data from the log. Filtering can be quickly enabled or disabled via the tool bar. In the free version, filtering is limited to patterns, however more patterns can be used by selecting enable in the Free Edition Limited Feature dialog.
  • Transformations
    Limited in Free Edition
    Transformations are used to reformat the text that is displayed in the log. A pattern is used to match lines in the log that are then substituted with the Replace template. Transforms are very flexible allowing reordering of text, changes in case, "swizzling" of characters, and error code lookup and substitution. Transforms can be quickly enabled or disabled via tool bar. In the free version, transforms are limited to patterns, however more patterns can be used by selecting enable in the Free Edition Limited Feature dialog.
  • Actions
    Limited in Free Edition
    Actions allow custom behaviour to be executed when a particular line in the log file is double clicked. For example, a pattern that matches a file path and line number can be associated with an action that opens the file at the given line number in a text editor, a useful feature when viewing application debug output. In the free version, actions are limited to patterns, however more patterns can be used by selecting enable in the Free Edition Limited Feature dialog.
  • Scratch Pad for Pattern Testing
    The patterns that are at the core of many of the features in RyLogViewer can become very complex. To aid in ensuring a pattern behaves correctly, the pattern editor features a test area that allows the behaviour of a pattern to be previewed. This test area is a useful tool for learning regular expression syntax.
  • Aggregate Files
    Licensed feature
    The aggregate files feature allows multiple files to be viewed as though they are a single file. All features (including highlighting, searching, filtering, etc) operate on the aggregate file in the same manor as they do for a single file. This is particularly useful for systems that produce sequentially numbered log files.

    In addition, the Export function can by used to create a single amalgamated file that is the result of all of the files concatenated together after filtering and/or text transformations have been applied.
  • Android Logcat output
    One of the targeted uses for RyLogViewer is to aid Android application development. The Android operating system has a system wide logging feature that gives great insight into the applications running on the device. The Android Logcat Wizard simplifies the process of connecting and capturing this logcat data.
  • Program Output Logging
    The Log Program Output dialog can be used to launch another application and capture its standard output or standard error.
  • Serial Port Logging
    Embedded systems frequently use RS232 serial comms as a simple means of transmitting internal state to the outside world. RyLogViewer's serial port logging is ideally suited to this scenario as it can receive data directly from a serial port and capture it into a file. Highlighting can then be used to make errors stand out, or state machine states/transitions obvious. The transforms feature can be used to replace error codes with meaningful error messages.
  • TCP/UDP Network Connection Logging
    The Log Network Connection dialog allows the capture of log data streamed over a network using either TCP or UDP protocols. The connection can be direct or via a proxy server.
  • Named Pipe Logging
    Named pipes are a common form of inter-process communication. Writing log output to a named pipe is another method of getting log data out of an application or server. The Log Pipe Connection dialog allows RyLogViewer to connect directly to the named pipe and capture any output received.
  • Export
    The Export option, under the file menu, writes the current log data to a file applying all enabled filters and transforms. Exporting is therefore a flexible way to transform a file and save the result. Exporting can also be performed by running RyLogViewer from the command line which allows RyLogViewer to be useful in batch scripts.
