Command Line Syntax
﻿
Syntax:
RyLogViewer [filepath] [options]

Following are the valid command line options for RyLogViewer. These options may be given in any order. Note that there is no whitespace between options and their associated parameter.
OptionMeaning
"file path"
The path to the file to load on startup.

Example:
RyLogViewer "mylogfile.log"
-p
Run the application in portable mode.

In this mode the application can run from any location that allows file write access, e.g. a USB drive. Settings are written to the directory that contains the application. If a file with the name 'portable' is found in the same directory as the executable then portable mode will be enabled by default.

Example:
RyLogViewer ‑p
‑s"settings file path"
Loads RyLogViewer using the specified settings file path.

This option is useful for running multiple instances of the application without overwriting the settings of other instances.

Example:
RyLogViewer ‑s"C:\temp\mysettings.xml"
‑e"export file path"
Export the "file path" argument with filter and transform patterns applied.

When exporting, a temporary copy of the settings file is used (either the default settings, or the file specified using the ‑s argument). This means that the settings file is not modified by other command line options (‑ps, ‑rdelim, ‑cdelim) when the ‑e export options is used.

The application main window is not displayed during the export and, on completion, the returned exit code is 0 for success, 1 for failure.

Examples:
RyLogViewer ALogFile.log ‑e"C:\exported logs\ALogFile.Exported.txt"
RyLogViewer ALogFile.log ‑ps"my_set.pattern_set" ‑e"C:\exported logs\ALogFile.Exported.txt"
‑ps"pattern set file path"
Loads the specified pattern set.

Example:
RyLogViewer ‑ps"my_pattern_set.pattern_set"
‑rdelim"row delimiter"
The character sequence that separates rows.

When exporting, this option overrides the row delimiter in the loaded settings. If not exporting, this option updates row delimiter in the loaded settings to the given value. Use the following special strings to represent carriage return, line feed, and tab:
  • <CR> - carriage return
  • <LF> - line feed
  • <TAB> - tab
Example:
RyLogViewer ALogFile.log ‑e"C:\exported logs\ALogFile.Exported.txt" ‑rdelim"<CR><LF>"
‑cdelim"column delimiter"
The character sequence that separates columns.

When exporting, this option overrides the column delimiter in the loaded settings. if not exporting, this option updates the column delimiter in the loaded settings to the given value. Use the following special strings to represent carriage return, line feed, and tab:
  • <CR> - carriage return
  • <LF> - line feed
  • <TAB> - tab
Example:
RyLogViewer ALogFile.log ‑e"C:\exported logs\ALogFile.Exported.txt" ‑cdelim"<TAB>"
‑nogui
Suppresses the displaying of any GUI (e.g. progress dialogs) while running from the command line.
‑silent
Suppresses output to stdout during export while running from the command line.
‑l"logging file path"
RyLogViewer has diagnostic logging built in but disabled by default. By specifying a log file path logging can be enabled. This is mainly useful for diagnosing issues with RyLogViewer.

Example:
RyLogViewer ‑l"C:\temp\rylogviewer.log"
‑h or /?
Displays this help message.
