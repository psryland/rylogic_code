Filtering Tab
Filtering allows log data that is not currently relevant to be hidden. Typical use cases for filtering are; to show only the error messages in a log, or hide all log entries of a particular format. The order that filtering rules are applied is defined by the order that the patterns are listed. This allows a complex set of rules for keeping or rejecting lines from a log file to be constructed.
  1. Pattern Editor
    The pattern editor is used to create and modify patterns that will match lines in the log data. The pattern editor is described in greater detail here.
  2. Active Patterns
    It is sometimes convenient to disable a pattern without deleting it. Filtering patterns can be enabled or disabled by clicking in this column
  3. Filter Type
    A filter of type Keep will prevent a log line from being hidden, where as a filter of type Reject will prevent the log line from being displayed. Filters are applied in the order listed so, depending on the order of filters and the state of the Reject all rows by default check box, a complex set of rules for keeping and rejecting log data can be created.
  4. Expression
    This column displays the text of the filter pattern.
  5. Filter Pattern Editing
    Clicking the icon in this column loads the pattern into the pattern editor to be modified.
  6. Filtering Pattern Sets
    Pattern sets are used to save and load groups of patterns. This is useful when RyLogViewer is used frequently for different types of logs. All of the filter patterns can be easily switched by selecting a pattern set from the drop down menu.
